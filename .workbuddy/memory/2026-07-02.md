## 修复 ShrikeDxf.vcxproj.filters 缺少 Manager 过滤器
- 现象：VS 解决方案资源管理器里看不到 `ShrikeDxf/Manager` 目录（点不进去），但磁盘上 `D:\shrikedxf\ShrikeDxf\Manager` 实际存在 31 个 .h/.cpp。
- 根因：`ShrikeDxf.vcxproj.filters` 没有为 `Manager` 注册任何 `<Filter>` 节点，所有 Manager 文件都被挂到了 `Header Files\Dxf / Header Files\Common / Source Files\Dxf / Source Files\Common` 几个不含 "Manager" 字样的过滤器下，于是 VS 没有显示 `Manager` 虚拟目录。
- 处理：
  1. 在 `<ItemGroup>` 过滤器声明区追加三个节点：`Header Files\Manager`、`Source Files\Manager`、`Form Files\Manager`（GUID 随机生成）。
  2. 批量将 31 个 `Manager\*` 文件的 `<Filter>` 改为：`.h/.h(QtMoc)` → `Header Files\Manager`；`.cpp` → `Source Files\Manager`。
- 验证：31 个条目全部替换；用 `xml.etree.ElementTree` 解析文件，无语法错误。
- 涉及文件：`D:\shrikedxf\ShrikeDxf\ShrikeDxf.vcxproj.filters`
